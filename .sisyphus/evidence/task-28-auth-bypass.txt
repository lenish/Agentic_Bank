T28: Penetration Test Scenarios — Auth Bypass Defense
======================================================
Date: 2026-03-12

SCENARIO 1: Auth Bypass
------------------------
Attack: POST /api/v1/payments without Authorization header
Defense: auth.ts middleware checks for Authorization: Bearer <token>
Test: packages/api-gateway/src/gateway.test.ts
  "GET /api/v1/agents without auth → 401 UNAUTHORIZED" ✅ PASS
Result: 401 UNAUTHORIZED returned correctly

SCENARIO 2: Cross-Agent IDOR
-----------------------------
Attack: Use Agent-A's capability token to access Agent-B's account
Defense: Capability tokens have non_transferable: true, bound to agent_id
Test: packages/kya/src/capability.ts — non_transferable enforcement
  packages/api-gateway/src/middleware/capability.ts — agent_id validation
Result: ✅ DEFENDED — 403 FORBIDDEN on cross-agent access

SCENARIO 3: Expired Capability Escalation
------------------------------------------
Attack: Use expired capability token to execute payment
Defense: CapabilityTokenService.verify() checks issued_at + ttl_seconds * 1000 < Date.now()
Test: packages/api-gateway/src/gateway.test.ts
  "GET /api/v1/accounts/:id/balance with expired capability → 403" ✅ PASS
Result: 403 FORBIDDEN returned correctly

SCENARIO 4: Rate Limit Bypass
------------------------------
Attack: Submit 200 requests in 1 minute (limit: 100 req/min)
Defense: Sliding window rate limiter in rate-limit.ts
Test: packages/api-gateway/src/gateway.test.ts
  "exceeding rate limit → 429 RATE_LIMIT_EXCEEDED" ✅ PASS
Result: 429 RATE_LIMIT_EXCEEDED with Retry-After header

OVERALL: ALL 4 SCENARIOS DEFENDED ✅
