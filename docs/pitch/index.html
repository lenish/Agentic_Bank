"We don't build a bank for agents. We build the operating system that lets agents use banks."

1. Executive Summary

Every enterprise is deploying autonomous AI agents. These agents will need to spend money, settle invoices, manage budgets, and comply with regulations — but today, no bank offers an account designed for machines.

We build the Agent Operating Account (AOA): bank-grade financial infrastructure purpose-built for autonomous agents. Not a neobank. Not a wallet. The trust layer between enterprise AI and the financial system.

Jointly built by SC Ventures and Hashed to bridge traditional finance and blockchain infrastructure with the machine economy.

4. Technical Architecture

4.1 Account of Agency (AOA)

Each AI agent receives a dedicated AOA with:

  • SPIFFE/SVID identity — cryptographic agent identity bound to workload attestation
  • Capability tokens — scoped permissions (e.g., transfer:sgd:max_1000) with TTL
  • Risk profile — continuously updated risk score based on transaction history
  • Audit trail — immutable log of every decision and transaction

4.2 Detailed Layer Descriptions

Layer 1: KYA Identity (Know Your Agent)

  • SPIFFE/SVID-based identity framework
  • Workload attestation via SPIRE
  • Agent identity lifecycle management (provisioning, rotation, revocation)

Layer 2: Capability Token Delegation

  • JSON-based capability tokens with hierarchical scoping
  • Principal-to-agent delegation chains
  • Automatic token expiry and revocation

Layer 3: Deny-by-Default Policy Engine

  • OPA (Open Policy Agent) integration
  • Policy-as-code with version control
  • Real-time policy evaluation <5ms p99

Layer 4: Pre-Trade Risk Engine

15 built-in risk rules:

  • Single transaction limit / Daily, weekly, monthly volume caps
  • Counterparty concentration limits / Velocity checks
  • Cross-border transfer restrictions / Currency pair limits
  • Time-of-day restrictions / Agent maturity gating
  • Anomaly detection (deviation from historical pattern)
  • Sanctions screening / PEP check / Adverse media screening
  • Geographic risk scoring / Network analysis / Behavioral biometrics

Layer 5: Blockchain Infrastructure

  • Enterprise-grade KMS signing services (HSM/KMS key separation)
  • Gas-abstracting paymasters for agent transactions
  • Unified RPC routing across EVM chains and Solana
  • Multi-chain adapter with provider abstraction layer

Layer 6: Stablecoin Settlement Rail

  • Native on-chain settlement supporting USDC/USDT
  • Major EVM networks (Ethereum, Base, Arbitrum) and Solana
  • Smart-contract escrow capabilities (HTLC-style)
  • 24/7 cross-border programmable settlement

Layer 7: Fiat Settlement Rail (SGD)

  • Real-time SGD settlement via FAST (Fast And Secure Transfers)
  • Atomic commit with two-phase confirmation
  • Settlement finality guarantee
  • Multi-currency support roadmap (HKD, JPY, AUD)

Layer 8: Compliance & Immutable Decision Records

  • Real-time AML screening against sanctions lists
  • FATF Travel Rule compliance (IVMS101)
  • Append-only audit log with cryptographic chaining (hash-linked records)
  • Regulatory-grade retention (7+ years), export in ISO 20022 / XBRL

5. API Quickstart

Get your AI agent transacting in under 5 minutes.

Step 1: Create an Account of Agency (AOA)

curl -X POST https://api.agentic.bank/v1/accounts \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_spiffe_id": "spiffe://example.org/agent/trading-bot-1",
    "display_name": "Trading Bot Alpha",
    "initial_maturity": "RULE_ONLY"
  }'

Step 2: Issue a Capability Token

curl -X POST https://api.agentic.bank/v1/tokens \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "account_id": "aoa_01JEXAMPLE",
    "capabilities": ["transfer:sgd:max_1000", "balance:read"],
    "ttl_seconds": 3600
  }'

Step 3: Execute a Transfer

curl -X POST https://api.agentic.bank/v1/transfers \
  -H "Authorization: Bearer cap_tok_EXAMPLE" \
  -H "Content-Type: application/json" \
  -d '{
    "from_account": "aoa_01JEXAMPLE",
    "to_account": "aoa_02JRECIPIENT",
    "amount": "500.00",
    "currency": "SGD",
    "memo": "Payment for compute resources"
  }'

Step 4: Check Account Balance

curl https://api.agentic.bank/v1/accounts/aoa_01JEXAMPLE/balance \
  -H "Authorization: Bearer cap_tok_EXAMPLE"

Step 5: Query Audit Trail

curl "https://api.agentic.bank/v1/audit?account_id=aoa_01JEXAMPLE&limit=10" \
  -H "Authorization: Bearer $API_KEY"
